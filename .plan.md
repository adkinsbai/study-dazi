# 用户画像功能实现计划

## 一、功能概述

在学习路径生成前，通过详细的问卷收集用户信息，生成个性化的用户画像（.md格式），用于指导后续的路径生成。

---

## 二、表单字段设计

### 第一步：学习目的（必选）

**主问题：你为什么想学这个？**

选项（单选）：
- 🎯 找工作/转行
- 📚 期末复习/考试
- 🔬 做研究/学术
- 💡 兴趣爱好
- 💼 副业/创业
- 📈 技能提升/晋升
- 🎓 完成作业/项目

### 第二步：当前水平（必选）

**主问题：你目前对这个领域的了解程度？**

选项（单选）：
- 🌱 完全零基础（没接触过）
- 📖 看过一些教程/视频
- ✍️ 做过一些练习/小项目
- 🛠️ 有完整项目经验
- 💼 有工作经验/专业背景

**追问（根据选择动态展示）：**
- 如果选"看过教程"：看的什么教程？大概学到哪里了？
- 如果选"做过练习"：做过什么类型的练习？
- 如果选"有经验"：有多少年经验？做过什么项目？

### 第三步：时间安排

**主问题：你打算用多少时间来学习？**

- 每周可用时间：`___` 小时/周
- 学习期限：`___` 个月（可选填）

### 第四步：深层目标（关键！）

**主问题：学完之后，你具体想做什么？**

这是一个开放性问题，但提供一些引导选项：
- 想进什么类型的公司？（大厂/外企/国企/创业公司/远程工作）
- 想做什么类型的项目？（Web应用/移动App/数据分析/自动化/游戏）
- 想达到什么水平？（能独立完成项目/能通过面试/能教别人/了解原理即可）
- 有没有具体的目标？（拿到某个offer/完成某个项目/通过某门考试）

### 第五步：感兴趣的方向（多选）

**主问题：在"领域"中，你对哪些方向比较感兴趣？**

（根据用户输入的领域动态生成选项，例如前端开发）：
- ⚛️ React 生态
- 🖼️ Vue 生态
- 🎨 UI/UX 设计
- 📱 移动端开发
- 🚀 性能优化
- 🔧 工程化/DevOps
- 🌐 3D/可视化

### 第六步：学习风格偏好

**主问题：你更喜欢哪种学习方式？**

（多选，最多选2个）
- 📹 视频教程
- 📝 文字教程/文档
- 🛠️ 实战项目驱动
- 📖 理论深入理解
- 🤝 社区交流/找人带

### 第七步：背景信息（可选但推荐）

**当前职业/专业：**
`___`（例如：计算机专业大三/产品经理/自由职业）

**已有相关技能：**
`___`（例如：会一点Python/学过C语言/会用Excel）

**学习动机强度：**
- 🔥 非常急迫（1-2个月内要用）
- ⏰ 比较急迫（3-6个月内要用）
- 🌊 不急（慢慢学，没有硬性deadline）

## 三、用户画像 .md 模板

```markdown
# 用户学习画像

## 基本信息
- **学习领域**: 前端开发
- **生成时间**: 2026-06-02

## 学习目的
**主要目标**: 找工作/转行

**深层目标**:
- 目标公司类型: 互联网大厂
- 目标项目类型: Web应用
- 期望水平: 能独立完成项目并通过面试
- 具体目标: 6个月内拿到前端开发offer

## 当前状态
**水平评估**: 看过一些教程/视频

**详细背景**:
- 职业/专业: 产品经理，想转行
- 已有技能: 会一点HTML/CSS，了解基本概念
- 学习经历: 看过B站的入门教程，做过几个小demo

## 学习条件
**时间安排**:
- 每周可用: 15小时
- 学习期限: 6个月
- 时间段: 工作日晚上 + 周末

**动机强度**: 🔥 非常急迫

## 学习偏好
**感兴趣方向**:
- ⚛️ React 生态（主要）
- 🎨 UI/UX 设计（次要）
- 🚀 性能优化（了解）

**学习风格**:
- 🛠️ 实战项目驱动（首选）
- 📹 视频教程（辅助）
- 📝 文字文档（查阅）

## 画像摘要
> 一位产品经理想转行前端开发，目标是6个月内进大厂。有基础的HTML/CSS知识，
> 希望通过实战项目驱动学习React生态，每周能投入15小时。学习动机强，
> 需要系统化的路径，重点是能通过面试的实战能力。
```

## 四、AI Prompt 设计

### 用户画像生成 Prompt

```
你是一位资深的学习规划顾问。根据用户填写的问卷信息，生成一份详细的学习画像。

输入信息：
- 学习领域：{domain}
- 学习目的：{purpose}
- 当前水平：{level}
- 水平详情：{levelDetail}
- 每周时间：{hoursPerWeek}
- 学习期限：{deadline}
- 深层目标：{deepGoal}
- 感兴趣方向：{interests}
- 学习风格：{learningStyle}
- 背景信息：{background}
- 动机强度：{urgency}

请生成一份 Markdown 格式的用户学习画像，包含：

1. **基本信息**：领域、生成时间
2. **学习目的**：主要目标、深层目标（拆解用户填写的内容）
3. **当前状态**：水平评估、详细背景（整理用户的背景信息）
4. **学习条件**：时间安排、动机强度
5. **学习偏好**：感兴趣方向、学习风格
6. **画像摘要**：用2-3句话总结这个用户的核心特点和学习需求

要求：
- 语言简洁专业
- 对用户输入进行合理推断和补充（例如根据"产品经理转行"推断可能有产品思维优势）
- 画像摘要要突出个性化特点，为后续的路径生成提供明确指导
- 输出纯 Markdown，不要代码块包裹
```

### 路径生成 Prompt 升级

在原有的 `FRAMEWORK_PROMPT` 基础上，增加用户画像作为上下文：

```
你是一位资深课程设计师。根据用户画像和学习需求，生成个性化的学习路径。

【用户画像】
{userProfile}

【学习需求】
- 领域：{domain}
- 时间：{hoursPerWeek}小时/周

请根据用户画像中的：
- 学习目的和深层目标 → 设计有针对性的学习阶段
- 当前水平 → 合理设置起点难度
- 感兴趣方向 → 在可选节点中体现偏好
- 学习风格 → 调整理论vs实践的比例
- 时间安排 → 合理估算每个阶段的时间

返回纯JSON...
```

## 五、技术实现方案

### 5.1 新增文件

1. **表单组件**: `src/components/path/user-profile-form.tsx`
   - 多步表单，7个步骤
   - 使用现有的 UI 风格（Tailwind + useState）
   - 支持前进/后退/跳过

2. **用户画像展示**: `src/components/path/user-profile-viewer.tsx`
   - 渲染 .md 内容为 HTML
   - 提供原始 .md 查看切换

3. **API - 生成画像**: `src/app/api/paths/generate/profile/route.ts`
   - 接收表单数据
   - 调用 AI 生成用户画像
   - 返回 .md 内容

4. **Prompt 文件**: 更新 `src/lib/path-prompts.ts`
   - 添加 `PROFILE_PROMPT`
   - 升级 `FRAMEWORK_PROMPT` 支持用户画像

### 5.2 修改文件

1. **路径创建页面**: `src/app/(dashboard)/paths/new/page.tsx`
   - 在原有表单前增加用户画像步骤
   - 新的 Step: `'profile' | 'intent' | 'framework' | 'nodes' | 'ready'`
   - 生成的用户画像传递给后续步骤

2. **数据库 Schema**: `prisma/schema.prisma`
   - `LearningPath` 增加 `userProfile` 字段（String?）
   - 存储用户画像 .md 内容

### 5.3 流程变更

```
原流程: 输入领域/水平 → 生成框架 → 展开节点 → 保存

新流程:
1. 填写详细问卷（7步）
2. AI生成用户画像（.md）→ 展示给用户确认
3. 基于画像生成学习框架
4. 展开节点
5. 保存（包含用户画像）
```

## 六、开发阶段展示

在开发阶段，用户画像会：
1. 在生成后立即展示完整的 .md 内容
2. 在路径详情页增加"查看画像"按钮
3. 在控制台打印生成的画像内容（调试用）
